# Contributing to InkPDF

Thanks for your interest in contributing! Here's how to get started.

## Development Setup

1. Fork and clone the repo
2. Open `index.html` in your browser (no build step needed)
3. Make your changes
4. Test across different PDF types (text-heavy, image-heavy, multi-page, scanned)
5. Submit a pull request

## Guidelines

- Keep it a single HTML file. The zero-config simplicity is a core feature.
- All external libraries should be loaded via CDN.
- Maintain the dark theme aesthetic and design language.
- Test on Chrome, Firefox, and Safari before submitting.
- Keep PRs focused on a single feature or fix.

## Reporting Issues

Open a GitHub Issue with the following details.

- Browser and version
- Steps to reproduce
- Expected vs actual behavior
- Screenshot if applicable
- Sample PDF (if the issue is file-specific, and only if it does not contain sensitive data)

## Feature Ideas

Check the README for the current wishlist. If you want to work on something not listed, open an issue first to discuss the approach.
